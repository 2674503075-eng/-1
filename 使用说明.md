# 局域网聊天（服务器端使用说明 / Windows PowerShell）

本说明重点覆盖：如何启动服务、如何彻底关闭并释放端口、如何确认服务是否正常、数据文件位置与清理方式、常见报错处理。

---

## 1. 启动服务器

### 方式 A（推荐）：运行后端入口

在项目根目录打开 PowerShell，执行：

```powershell
py -3 backend/app.py
```

成功标志：看到类似输出：

```text
Uvicorn running on http://0.0.0.0:8000
```

### 方式 B：用 uvicorn 启动

```powershell
py -3 -m uvicorn backend.app:app --host 0.0.0.0 --port 8000
```

注意：不要在 `--port 8000` 后面加反斜杠 `\`，否则会报：

```text
Invalid value for '--port': '8000\' is not a valid integer.
```

---

## 2. 访问方式（局域网）

- 本机访问：`http://localhost:8000/`
- 局域网其他设备访问：`http://你的服务器IP:8000/`
  - 示例：`http://192.168.110.134:8000/`

查看本机 IP（选一个局域网网卡的 IPv4）：

```powershell
ipconfig
```

---

## 3. 彻底关闭服务器（释放端口 8000）

### 方法 1：在启动服务器的 PowerShell 窗口直接停止

在运行服务器的窗口按：

```text
Ctrl + C
```

### 方法 2：不知道哪个进程占用 8000（强制停服）

1) 找出占用 8000 的 PID：

```powershell
netstat -ano | findstr :8000
```

你会看到类似：

```text
TCP    0.0.0.0:8000    0.0.0.0:0    LISTENING    12345
```

其中 `12345` 就是 PID。

2) 强制结束该 PID：

```powershell
taskkill /F /PID 12345
```

3) 再次确认端口已释放（没有 LISTENING 即正常）：

```powershell
netstat -ano | findstr :8000
```

### 方法 3：一键“能杀就杀”（只要确保彻底停服）

```powershell
for /f "tokens=5" %a in ('netstat -ano ^| findstr :8000 ^| findstr LISTENING') do taskkill /F /PID %a
```

## 4. 验证服务是否正常

### 4.1 HTTP 是否正常（返回 200）

```powershell
powershell -NoProfile -Command "(Invoke-WebRequest -UseBasicParsing http://localhost:8000/).StatusCode"
```

输出 `200` 表示正常。

### 4.2 端口是否在监听

```powershell
netstat -ano | findstr :8000
```

看到 `LISTENING` 表示服务已运行。

## 5. 数据与文件存储位置（服务器端）

### 5.1 聊天记录（已统一放入独立文件夹）

聊天记录都在项目根目录下的 `history\` 文件夹中：

- 群聊公共池（如存在）：`history\history___public__.json`
- 每个用户独立文件（按 UID）：`history\history_<uid>.json`
  - 示例：`history\history_u_1773916848213_80199520.json`

### 5.2 上传文件目录

上传的文件位于：

- `uploads\`

说明：
- 文件撤回时会尝试直接删除上传文件；若删除失败，会改名为 `.revoked` 用于立刻失效下载链接。

---

## 6. 清空数据（恢复“出厂”）

建议先停服再清理。

### 6.1 仅清理聊天记录

删除 `history\` 目录下的所有 `history*.json` 文件。

### 6.2 清理聊天记录 + 上传文件

- 清空 `history\` 下所有历史文件
- 清空 `uploads\` 下所有文件

---

## 7. 常见问题排查

### 7.1 端口占用（WinError 10048）

报错类似：

```text
[winerror 10048] 通常每个套接字地址(协议/网络地址/端口)只允许使用一次
```

按“第 3 节：彻底关闭服务器”用 `netstat` + `taskkill` 处理即可。

### 7.2 麦克风权限（语音通话）无法获取

多数浏览器要求 **HTTPS 或 localhost** 才允许获取麦克风（`getUserMedia`）。

如果你是用 `http://192.168.x.x:8000` 打开的，可能会被限制。

---

## 8. 安全预警（红色提示）

服务器端遇到明显异常数据或文件 IO 异常，会输出红色 `[ALERT] ...` 提示，便于你在 PowerShell/终端快速定位问题。
